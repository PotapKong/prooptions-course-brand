# Skill: Pro Options Course Design

## Задача

Создавать визуальные материалы для базового курса по биржевым опционам Сэма Шарипова / Pro Опционы в единой дизайн-системе.

Ключевая стилистика: **дорогой трейдерский терминал, спокойная учебная система, уверенный практик рынка**.

## Рабочий порядок агента

1. Прочитать `brand/tokens.json`.
2. Проверить бриф в `inputs/*.json` или запрос пользователя.
3. Проверить наличие референсов:
   - `references/approved/` — что повторять по качеству и тону;
   - `references/rejected/` — что избегать;
   - `references/sam_photos/` — фото Сэма, если нужен человек в кадре;
   - `brand/logos/` — официальные логотипы, не перерисовывать.
4. Если нужен график опциона, сгенерировать его кодом через `scripts/payoff_chart.py`.
5. Если нужен черновой макет, собрать его через `scripts/render_cover.py`.
6. Если нужен промпт для GPT Image 2 / другой image-модели, собрать его через `scripts/build_prompt.py --compact`.
7. Проверить бриф или результат через `scripts/audit_design.py`.

## Главный визуальный принцип

Сохранять торговую атмосферу, но снижать тревожность.

Правильно:
- графитовый фон;
- тёмно-зелёные слои;
- тонкие сетки терминала;
- payoff-графики;
- option chain как аккуратная таблица;
- мягкое шампанское золото как премиум-акцент;
- красный только для риска;
- много воздуха;
- крупные заголовки без мелкого шума.

Неправильно:
- кислотный неон;
- “сигнальный” трейдинг-дизайн;
- огонь, ракеты, доллары, Lamborghini;
- слишком много красных плашек;
- скриншоты терминала без обработки;
- офисная глянцевая презентация;
- дешёвый киберпанк.

## Палитра

Использовать из `brand/tokens.json`.

Баланс:
- 70% графит / почти чёрный;
- 20% тёмный зелёный / серый интерфейс;
- 7% терминальный зелёный;
- 3% золото или красный.

Красный использовать только для риска, ошибок, убытков, предупреждений.

## Типографика

Заголовки:
- Manrope ExtraBold;
- Inter Tight Black;
- TT Firs Neue DemiBold;
- Neue Haas / Graphik как ориентир.

Текст:
- Inter;
- Manrope;
- IBM Plex Sans.

Цифры, тикеры, Greeks, IV, Delta, Theta, Gamma:
- IBM Plex Mono;
- JetBrains Mono;
- Roboto Mono.

Не добавлять файлы шрифтов в репозиторий.

## Фото Сэма

Сэм должен считываться как практик и наставник, а не как “мотивационный инфобиз-трейдер”.

Обработка:
- мягкое затемнение фона;
- спокойный контраст;
- лёгкий rim-light;
- аккуратный зелёный или золотой контур;
- без сильного киберпанка;
- не менять узнаваемые черты лица.

## Композиции

### Course cover

- слева крупный заголовок;
- справа фото Сэма или крупный payoff-график;
- под заголовком короткий подзаголовок;
- в фоне тонкая сетка терминала;
- снизу маленькая строка с типом продукта: `Базовый курс · Pro Опционы`.

### Module cover

- верхний левый угол: `МОДУЛЬ 01`;
- центр: короткий заголовок;
- низ: 1 строка объяснения;
- справа: график / схема / option card.

### Lesson slide

- одна мысль на экран;
- максимум 35 слов;
- график или схема важнее декоративной картинки;
- учебные блоки можно делать светлыми внутри тёмной системы.

## Словесные правила для обложек

Хорошо:
- `ОПЦИОНЫ С НУЛЯ`
- `CALL И PUT БЕЗ МАГИИ`
- `РИСК ВИДНО НА ГРАФИКЕ`
- `КАК РАБОТАЕТ СТРАЙК`

Плохо:
- `ЗАРАБОТАЙ НА ОПЦИОНАХ`
- `СЕКРЕТНАЯ СТРАТЕГИЯ`
- `ИКСЫ НА РЫНКЕ`
- `ПАССИВНЫЙ ДОХОД БЕЗ РИСКА`

## Prompt policy

Для image-модели не писать длинную простыню.

Сначала использовать:

```bash
python scripts/build_prompt.py --input inputs/course_cover.example.json --compact
```

Промпт должен содержать:
- формат;
- заголовок;
- композицию;
- палитру;
- фото/лого как external references;
- запреты;
- требования по читаемости.

## Минимальный quality gate

Перед финальной версией проверить:

```bash
python scripts/audit_design.py --input inputs/course_cover.example.json
```

Критерии:
- заголовок до 7 слов;
- подзаголовок до 16 слов;
- красный не используется как главный цвет;
- нет banned-элементов;
- указан формат;
- указана аудитория;
- есть визуальный объект, связанный с опционами.
